# Visual-Experience Audit — 2026-07-21

_Scope: current state of the project with focus on validity and consistency of the
visual experience (`web/office.css`, `web/index.html`, HTML generated by
`web/render.js` + `web/app.js`, visual test coverage, and design-process docs).
Method in §9. All line numbers verified against `main` @ `b435069`._

---

## 1. Executive summary

**Verdict: good bones, accumulating sediment — and one live crash bug.**

The design-token system is coherent (light/dark palettes, seasonal accents,
safe-area insets), typography is consistent and rem-based, and modern CSS
(`color-mix`, `:focus-visible`, `env()`) shows care. But:

- **A P0 crash bug breaks every day with an alternate observance** (page stuck
  on "Loading…"). Staging is in user testing pending promote — this blocks it.
- **~15% of `office.css` (~120 lines) is dead or unreachable**, sediment from
  two redesigns (nav layout, MP/EP control). Two token-level bugs are live.
- **The visual layer has no automated safety net**: no screenshot/visual
  regression testing, dark mode completely untested, CSS never parsed, E2E not
  in CI, and the one deploy gate that should run E2E (`make test-staging`)
  selects **zero tests**. A CSS break ships silently today.
- **ADR 0010 (design prototyping) is documented but not operating**: its one
  prototype page was never deleted, a second throwaway was committed inside a
  production commit, and every visual change after the MP/EP toggle skipped
  prototyping entirely.
- **Doc drift is significant**: all 10 ADRs remain "Proposed" though
  implemented; `docs/DESIGN.md` describes a Go app that no longer exists;
  `AGENTS.md` references a deleted file and a non-existent BUGS.md section.

P0 + five quick wins are specified for immediate implementation in **Appendix A**
(Commits 1–6). The remaining findings form the roadmap in §8.

## 2. P0 and validity bugs

### 2.1 P0 — `renderObservanceCard` throws `ReferenceError` on every alternate-observance day

- `web/app.js:536–551`: the function interpolates `esc(dateStr)` and
  `esc(officeType)` (lines 543, 549), but `dateStr`/`officeType` are parameters
  of `render()` (`app.js:766`), not in the function's scope. Confirmed by
  executing the verbatim function in Node: `ReferenceError: dateStr is not
  defined`.
- Called unconditionally at `app.js:993` (`let html =
  renderObservanceCard(officeData, activeObs)`). On any day whose lectionary
  office object has `.alternate` (e.g. 2026-05-17: `morning.alternate` and
  `evening.alternate` both present in `data/lectionary/2026-05.json`), the throw
  aborts `render()` before `contentEl.innerHTML = html` (`app.js:1094`) —
  `#office-content` stays on "Loading…" forever.
- **Masked from notice, not from effect**: the card itself is `display:none`
  (`office.css:768`, superseded by the header observance control in `ddf250e`),
  but the throw happens at string-build time, before CSS is relevant. The header
  control (`ctrlEl.innerHTML` at `app.js:897`) is inserted *before* the throw,
  so the page looks partially alive.
- Introduced 2026-06-06 in `cab06dd` ("feat: observance as content card") —
  broken since birth. Circumstantial confirmation:
  `test-results/.last-run.json` = `"status": "failed"` (Jul 18, immediately
  after `ddf250e` landed); the E2E observance fixture (2026-05-17) exercises
  exactly this path.
- **Fix: delete, don't repair.** The component is dead UI. Spec: Appendix A,
  Commit 1.

### 2.2 Duplicate ARIA IDs in every standard render

- `web/render.js:356`: `lessonHtml` calls `renderAlternatives(readingResponse,
  shared, 'reading_response')` with a **fixed contextKey**. Two lessons/day →
  IDs `pwc-alt-reading_response-tab-0…2` / `-panel-0…2` each appear **twice**
  (confirmed by executing `lessonHtml` twice: 12 IDs, 6 duplicates).
- `web/app.js:558`: `gloriaHtml` uses fixed contextKey `'doxology'`; emitted
  once per observance block (`app.js:1036–1048`) → duplicate `pwc-alt-doxology-*`
  IDs on days with an alternate observance.
- `aria-controls`/`aria-labelledby` reference ambiguous targets — invalid HTML,
  broken AT tab-panel association. `activateTab` (`app.js:1233–1250`) only works
  because it ignores IDs and uses shared `data-key`.
- Fix: Appendix A, Commit 2 (per-instance ID salt; `data-key`/localStorage keys
  unchanged so cross-lesson tab sync and persistence are preserved).

### 2.3 Undefined design token — silent wrong font

- `web/office.css:648`: `.fats-bio-body { font-family: var(--font-body); }` —
  `--font-body` is defined nowhere (only `--font-liturgy` and `--font-ui`
  exist). The declaration is invalid at computed-value time; the bio body
  silently inherits the UI font. Fix: Appendix A, Commit 3.

### 2.4 Write-only design token

- `--color-day` (`office.css:27`) is set by JS on every render (`app.js:857`)
  and on colour-chip cycling (`app.js:1361`), but **no CSS rule ever reads
  `var(--color-day)`**. Pure waste. Fix: Appendix A, Commit 4.

### 2.5 StatusBar style ternary with identical branches

- `web/app.js:73`: `isDark ? Style.Dark : Style.Dark`. The background is always
  dark green `#15382A` (`app.js:74`), and Capacitor `Style.Dark` = light content
  for dark backgrounds — always correct. Fix = delete the branch and the
  now-unused `isDark` line: Appendix A, Commit 5.

### 2.6 Dead deploy gate — `make test-staging` runs zero tests

- `Makefile:205–207` runs `npx playwright test --grep "@smoke"`; no spec
  contains an `@smoke` tag (only repo hit is the Makefile itself). The gate
  trivially passes while testing nothing. Fix: Appendix A, Commit 6.

### 3.2 Dark mode gaps

- No `prefers-color-scheme` fallback — dark is manual-toggle only
  (`app.js:128–134`). No `<meta name="color-scheme">`, no root `color-scheme`
  property → scrollbars/form controls don't adapt (see also 3.1 date input).
- **Seasonal accents are not dark-adjusted**: `[data-theme="dark"]`
  (`office.css:55–69`) redefines every color except `--color-accent`. Advent
  `#4B3F8C` and Passiontide `#7E2420` on `#15140E` are near-unreadable at
  0.82rem uppercase (`#day-office-name`). Christmas/Easter gold `#B0863A` on
  light cream `#F4EEDF` is ~2.5:1.
- Brass tokens one hex digit apart in dark theme — `--color-brass: #CBA24E`
  (line 67) vs `--color-brass-ink: #CBA24A` (line 68): probable typo; visually
  identical in dark, distinct in light.

### 3.3 Typography inconsistencies

- **Font weights used but not loaded**: `index.html:9` loads EB Garamond
  400/500/600 (+italic 400/500) and IBM Plex Sans 400/500. CSS uses Garamond
  700 (`.office-section-title:471`, `.scripture-ch-num:554`) and Plex 600
  (`.psalm-heading`, `.reading-heading`, `.day-ctrl-cap`, `.meta-lbl`) →
  browsers synthesize faux-bold.
- **Mobile line-height contradicts font-size scaling**: `office.css:70–72`
  hardcodes `.seg-leader/.seg-response { line-height: 1.7 }` under 520px,
  bypassing `--line-height`; with `data-font-size="large"` (`--line-height:
  1.75`, line 40) mobile users get 1.7 anyway.
- `--indent: 0` (line 7) is a constant — used at 449, 501, 507, 603, never
  reassigned. Dead abstraction.
- **Three divergent scripture/psalm/canticle conventions**: lessons get
  always-visible `<sup class="verse-num">`; psalms get bare `<sup>` hidden
  except in book mode (`office.css:558–559`); canticles get none. Body text in
  `.scripture-block` (multi-`<p>`, no pre-wrap), `.psalm-block` (single `<p>`,
  `<br>`-joined, pre-wrap), and `.seg-leader/-response` — each sets liturgy font
  at ~1.12–1.15rem independently rather than inheriting from `.liturgy`.
- **Three h3 styles at one outline level**: `.psalm-heading` (muted uppercase),
  `.reading-heading` (same + `border-top`, line 583), `.office-subsection-title`
  (liturgy font, green) — hierarchy between "Psalms:", "The Reading:", and
  subsections is arbitrary.
- Chapter drop-num renders even when a reading starts mid-chapter
  (`app.js:487`, `i === 0 ? chDrop`) — "Rom 8:18–25" opens with a large "8".
- **Two divergent implementations of the same tab component**:
  `renderAlternatives` (`render.js:287–302`, with `title` tooltip + 22-char
  truncation) vs `tabBlock` in `collectToggleHtml` (`app.js:661–671`, no
  tooltip, no truncation — long Occasional Prayer collect names can overflow).
- Attribution label flips: initial render "Translation: NRSVUE" (`app.js:1092`)
  vs after switching "Scripture: KJV" (`app.js:1201`) — same element, two labels.

### 3.4 Error/empty states

Styled consistently: `.loading`, `.error-msg`, `.out-of-range-*`,
`.scripture-offline`, `.scripture-fallback-note`. But:

- **Missing form** (`offices[key]` null, `app.js:836`): no message at all —

- **Unresolvable collect**: `collectHtml` emits a bare citation `p. 344` with

- Scripture failure has **three different visual treatments** depending on

- `CANTICLE_SOURCE` gaps and missing `reading_response` → `console.warn` only

## 4. Accessibility gaps (open)

- **Settings sheet** (`index.html:35–44`, `office.css:241–263`): "closed" via
  `pointer-events:none` + off-canvas transform only — its `<select>`/`<button>`s
  remain in the tab order while the container has `aria-hidden="true"` (WCAG
  violation). `role="dialog"` lacks `aria-modal="true"`; no focus trap, no
  focus return (Escape handled, `app.js:1330`). Fix: `inert` or
  `visibility:hidden`/`display:none` when closed + focus management.
- `#day-office-name` is a clickable `<div>` (MP/EP toggle, `app.js:1407–1410`,
  `cursor:pointer` at `office.css:347`) — no `role="button"`, no `tabindex`, no
  key handler. Its `::after` "▾" (`office.css:350–354`) advertises a dropdown
  that doesn't exist.
- No `prefers-reduced-motion` handling despite several transitions (settings
  sheet 0.28s slide, body color transition, colour-chip scale hover).
- Light-mode `--color-muted #7B7460` on `--color-bg #F4EEDF` ≈ 4.0:1 — below
  4.5:1 for the 0.72–0.78rem meta text it colors.
- Tabs: no roving tabindex (all tabs tabbable), no Home/End, tablists lack
  `aria-label`, `role="tabpanel"` panels lack `tabindex="0"`.
- Decorative `◆` in `.day-ctrl-obs-mark` (`app.js:889`) not `aria-hidden`;
  Latin antiphon names (`app.js:929–935`) lack `lang="la"` (see also UX-17).
- Print: `@media print` (816–824) does not hide the settings sheet (off-screen
  via transform only) — risk of stray print artifacts.
- Unescaped psalm title interpolation (`app.js:368` — `data.number`/`data.title`
  raw while every other dynamic string passes through `esc()`): escaping-
  consistency hole (data is local/trusted, so low severity).
- Positive: `:focus-visible` outlines, landmarks (`nav`/`main`/`header`),
  logical h1→h2→h3, rem-based sizing, safe-area insets, ARIA tablist pattern
  with arrow keys, `aria-expanded`/`aria-hidden` toggling.
