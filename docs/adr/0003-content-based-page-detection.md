# ADR 0003: Detect office page bounds from content, not hardcoded numbers

## Status
Proposed

## Context
The extraction pipeline must know which PDF pages contain each of the 31 Daily
Office forms. The page numbers depend on the PDF edition — a re-typesetting by
the publisher shifts every page boundary. The pipeline also needs page ranges
for BAS collects and fallback pages where font encoding breaks.

Hardcoding page numbers bakes a specific PDF edition into the extraction code.
A new edition silently produces garbage with no error, because the old page
ranges now point to unrelated content.

## Decision
Detect office page bounds from **text content** rather than from hardcoded page
numbers. A new tool `detect_office_bounds.py`:

1. Runs `pdftotext -layout` on the full PDF.
2. Scans each page for office form title patterns. The canonical patterns are
   multi-line aware and case-sensitive:

   | Season | Pattern |
   |---|---|
   | Advent | `Morning Prayer for Advent\n` or `Evening Prayer for Advent\n` |
   | Christmas | `Morning Prayer for Christmas\n` or `Evening Prayer for Christmas\n` |
   | Epiphany | `Morning Prayer for Epiphany\n` or `Evening Prayer for Epiphany\n` |
   | Lent | `Morning Prayer for Lent\n` or `Evening Prayer for Lent\n` |
   | Passiontide | `Morning Prayer for Passiontide\n` or `Evening Prayer for Passiontide\n` |
   | Easter | `Morning Prayer for Easter\n` or `Evening Prayer for Easter\n` |
   | Pentecost | `Morning Prayer for Pentecost\n` or `Evening Prayer for Pentecost\n` |
   | All Saints | `Morning Prayer for All Saints\n` or `Evening Prayer for All Saints\n` |
   | Ordinary Sunday | `Morning Prayer: Ordinary Time — Sunday\n` or `Evening Prayer: Ordinary Time — Sunday\n` |
   | Ordinary Monday–Saturday | Same pattern with weekday name |

3. Validates: must find exactly 31 forms (16 seasonal + 15 Ordinary Time).
   Reports which forms were found and which were missing.
4. Computes the end page for each form as the page before the next form's start
   (or the last office page for the final form).
5. Writes `tools/office_bounds.json` — a committed JSON file:
   ```json
   {
     "advent-mp": {"start": 14, "end": 21},
     "advent-ep": {"start": 22, "end": 28},
     ...
   }
   ```

The extraction tools read this file instead of a hardcoded table.

### Handling a new PDF edition
When the PDF is re-typeset:
1. Run `detect_office_bounds.py --strict` — the new page numbers cause
   `office_bounds.json` to disagree with detected bounds. The tool reports
   the delta and exits non-zero.
2. Run `detect_office_bounds.py --write` — regenerates `office_bounds.json`.
3. Commit the updated file.

### False-positive guard
Form titles may appear in a table of contents on earlier pages. The detector
requires that a title match be the **first substantial text on the page**
(within the top 15% of the page height, excluding running headers). This
distinguishes body-form titles from TOC entries, which appear in dense
lists of text.

The same content-based approach applies to collect page ranges and fallback
pages in the BAS extraction.

## Consequences

### Positive
- Survives PDF re-typesetting: one command regenerates the bounds file.
  No code change required.
- Detection failures produce specific error messages identifying which forms
  were found and which were missing.

### Negative
- If the publisher changes form title wording (e.g., "Morning Prayer during
  Advent"), the canonical patterns must be updated. The `--strict` failure
  message includes the actual text found at each candidate position, making
  the update mechanical.
- Initial setup requires running `detect_office_bounds.py --write` against the
  current PDF to seed the bounds file.

### Notes
- The committed JSON file is regenerated by the detector, not manually edited.
  It serves as a cache and a source of truth for CI — `--strict` validation in
  CI catches drift between the cached bounds and the current PDF.
